Making it reusable across verticals

Only one file is brand-specific

src/generator.js holds the product catalog, AOV ranges, and event properties. Everything else — the MCP interface, Klaviyo client, rate limiting — is completely generic.

Swap the generator to change verticals

Replace PRODUCTS, AOV ranges in SEGMENT_CONFIG, and _eventProps(). Same MCP tools, same Claude prompts, different brand data.

Scenarios shape the RFM mix

Partners can pick standard, growth, or churn_risk to demonstrate different Klaviyo use cases without changing any code.

Clean resets between demos

Every seeded profile is tagged _seeder: true. reset_sandbox deletes only those — real account data is never touched.

Setup in 3 steps

1

Install & configure

npm install then add your Klaviyo private API key to config.json or set KLAVIYO_API_KEY as an env var. Key needs: profiles, lists, events read/write scopes.

2

Register with Claude Desktop

Add one entry to claude_desktop_config.json pointing at src/index.js. Restart Claude Desktop. The seeder tools appear automatically.

3

Tell Claude to seed the account

Say "seed this Klaviyo account with 250 profiles, standard scenario." Claude calls the tools, logs progress, and your sandbox is ready in minutes.

The pitch: Partners spend hours manually setting up demo accounts. This MCP server turns that into a single sentence to Claude. It's reusable across any Klaviyo account, extensible to any vertical, and resets cleanly between demos — so partners can focus on the story, not the setup.
